# Remove items from a collection in Workfront Library

You can remove assets and folders from any collection to which you have Manage permissions.

<ol> 
 <li value="1"> In Workfront, click the Main Menu icon , then select Library to open Workfront Library in a new browser tab. </li> 
 <li value="2"> <p>In the upper-left corner of Workfront Library, click the <span class="bold">Menu</span> icon <img src="assets/library-menu-icon.png">.</p> </li> 
 <li value="3"> <p>Click <span class="bold">Collections</span>, then open the collection from which you want to remove an item.</p> <p>(Conditional) If you are in the list view, click the <span class="bold">Show grid view</span> icon <img src="assets/grid-view-icon.png"> to switch to the grid view.</p> </li> 
 <li value="4"> <p>Hover over the item you want to remove, click the <span class="bold">More</span> icon in the upper-right corner of the item, then select <span class="bold">Remove from Collection</span> in the drop-down menu.</p> <p> <img src="assets/remove-from-collection-350x315.png" style="width: 350;height: 315;"> </p> <note type="tip">
   When you use the Remove from Collection option, the item is removed from the collection, but remains in Workfront Library. Only a Workfront Library administrator can delete content from Workfront Library. For information on deleting assets, see 
   <a href="../../../workfront-library/content-management/delete-items.md" class="MCXref xref">Delete items from Workfront Library</a>
  </note> </li> 
 <li value="5"> <p>Click <span class="bold">Remove</span> to confirm you want to remove the item.</p> <p>The item is removed from the collection. </p> <p>If the item you removed was used for the cover image on the collection, it remains as the cover image even though the item is no longer in the collection.</p> </li> 
</ol>
